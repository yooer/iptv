# IPTV 调度与 API 代理系统

本系统是一个高性能、轻量级的 IPTV 业务调度与后端 API 代理中转平台。采用 Go 语言开发，具备跨平台运行能力，支持大规模并发请求。

## 核心功能

### 1. IPTV 直播流调度
*   **多源负载均衡**：支持多个直播源服务器（如 TamronOS, MistServer, MediaMTX）同时接入。
*   **加权随机调度**：支持 1-10 权重设置，根据服务器性能分配流量。
*   **架构适配**：自动识别并适配不同流媒体服务器的播放地址格式。
*   **健康检测**：集成 Ping 检测模块，实时查看节点网络延迟。

### 2. 后端 API 反向代理
*   **集群转发**：将前端请求透明转发至后端 API 节点。
*   **故障转移**：支持主从节点配置，自动屏蔽不可用节点。
*   **路径优化**：自动处理 URL 路径中的多余斜杠（如 `//api/`），提升兼容性。

### 3. 管理后台
*   **可视化操作**：基于 Gin 框架构建的 Web 管理界面。
*   **节目映射**：维护频道 ID 与播放地址的映射关系，支持批量导入/导出节目单。
*   **系统监控**：实时查看数据库连接状态、系统运行版本等信息。

### 4. MediaMTX 集群管理 (NEW)
*   **一键自动化部署**：通过 SSH 远程自动安装、配置并挂载 MediaMTX 为系统服务。
*   **安全加固**：新增节点自动生成随机 API 凭据，防止默认弱口令风险。
*   **实时态势感知**：基于 SSE 技术实现安装日志实时回显，监控 HLS 会话连接。
*   **离线故障雷达**：频道管理支持“离线优先”排序与过滤，快速定位流异常。

### 5. 生产级推流护城河与高并发调优矩阵 (v1.4 升级)
系统在推流引擎内核层面，针对高帧率（如 1080p 50fps/50tbr）高并发转推场景，实装了全方位的金标准防丢包参数矩阵：
*   **50MB 底层网络发送缓冲 (`-buffer_size 52428800`)**：针对高帧流冷启动瞬间高达实时码率 3 倍以上的切片爆发吞吐量，为底层 TCP 套接字单独分配 50MB 级发送窗口，彻底解决高并发瞬间系统套接字打满导致的 `Broken pipe` 管道断连。
*   **4096 深度多路复用缓冲池 (`-max_muxing_queue_size 4096`)**：为打包流水线扩容 4 倍缓冲深度，从容吸纳突发 TS 包积压。
*   **时钟平滑与时间戳强制对齐**：采用原生单调时钟探测（`+genpts+discardcorrupt`），辅以音频时间轴自适应重采样（`aresample=async=1`）及输出协议级时间戳强制清零连续控制（`-copyts -start_at_zero -fps_mode passthrough`），确保下游端播放无缝秒开且永不音画脱节。

## 技术栈

*   **后端**: Go 1.22+ / Gin Web Framework
*   **通信**: SSE (Server-Sent Events) 实时日志流
*   **数据库**: MongoDB
*   **前端**: Vanilla CSS / HTML5 / FontAwesome 6 (由 `embed` 打包至二进制文件)
*   **安全**: 双重 MD5 加盐加密、Session TTL 机制、动态随机凭据生成

## 安装与部署

> [!IMPORTANT]
> 生产环境部署（含 Linux Systemd 守护进程配置）请参考：[详细部署手册 (DEPLOY.md)](./DEPLOY.md)

### 环境要求
*   **操作系统**: Windows / Linux / macOS
*   **数据库**: MongoDB 4.0+

### 快速启动
1.  **准备配置文件**: 系统首次运行会自动生成 `config.ini`。
2.  **启动程序**:
    ```bash
    # 直接运行
    ./iptv
    
    # 指定端口运行
    ./iptv --port 8081 --cmd run
    ```
3.  **初始化**:
    *   访问 `http://localhost:8081/dash/setup-page` 配置数据库连接。
    *   **自动初始化**: 系统会自动创建必要的唯一索引，并生成默认管理员账号。

### 默认凭据
*   **账号**: `admin@iptv.com`
*   **密码**: `123456`
*   *建议登录后立即前往系统设置修改密码。*

## 数据库自动维护机制

系统内置了自愈与初始化逻辑（`ensureDatabaseDefaults`），在每次连接数据库时会自动执行：

1.  **索引确保**:
    *   `Members`: `mail` (唯一索引)
    *   `proxy_api`: `ip` (唯一索引)
    *   `proxy_m3u8`: `ip` (唯一索引)
    *   `proxy_menu`: `id` (唯一索引)
2.  **账号兜底**: 若检测到用户表为空，将自动生成默认管理员账号。

## 编译指南

若需自行编译，请确保已安装 Go 环境：

```bash
# 编译 Windows 版本
go build -o iptv.exe main.go

# 编译 Linux 版本 (AMD64)
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -o iptv main.go
```
